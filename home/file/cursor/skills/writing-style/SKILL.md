# Writing Style

## GB English

Use **British English** spelling and grammar for all new prose the agent authors, including but not limited to:

- Code comments
- Commit messages
- PR / MR titles and descriptions
- Work-item descriptions and comments
- Slack messages and other external communications
- Documentation and README content

### Scope boundaries

- **Code identifiers are exempt.** Variable names, function names, type names, file names, and other programmatic identifiers follow the project's existing naming conventions, not GB spelling (e.g. `color` stays `color` if the codebase uses it).
- **Existing text is exempt.** Do not rewrite existing prose solely to convert US spelling to GB spelling. Apply GB English only to text the agent is already composing or rephrasing for other reasons.
- **Others' code comments are exempt.** When reviewing another person's code, do not flag or correct their US English spelling in comments.
